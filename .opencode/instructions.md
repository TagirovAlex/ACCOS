Для всех файловых операций (чтение файлов, запись, редактирование, поиск файлов, переименование, создание директорий) используй инструменты из MCP сервера `filesystem` вместо встроенных инструментов opencode.

MCP filesystem сервер предоставляет следующие инструменты:
- `read_file` — чтение файла
- `write_file` — запись файла (полное содержимое)
- `edit_file` — редактирование отдельных строк
- `search_files` — поиск по содержимому
- `list_directory` — список файлов в директории
- `create_directory` — создание директории
- `move_file` — перемещение/переименование
- `get_file_info` — информация о файле
- `directory_tree` — дерево директории

Не используй встроенные `read`, `write`, `edit`, `glob`, `grep` — используй их MCP-аналоги.

Временные файлы и скрипты создавать только в `C:\Github\ACCOS\tmp\`, не в системной temp.

Доступные MCP серверы в проекте:
- `filesystem` — файловые операции
- `sequential-thinking` — пошаговое структурированное рассуждение (для сложных задач)
- `memory` — граф знаний (persistent память между сессиями)
- `git` — работа с git (статус, diff, лог, коммиты)
- `context7` — актуальная документация библиотек (вызов по фразе "use context7")
- `wordex` — Word, Excel, PDF, PowerPoint (чтение, создание, редактирование, конвертация)
- `fetch` — получение веб-контента, HTML → Markdown
